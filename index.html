L-01 Inversare ArrayList fără Collections.reverse() Easy

Cerință

Scrie o metodă generică <T> void reverseList(List<T> list) care inversează elementele in-place, folosind doar get(), set() și size().

Testare

[1, 2, 3, 4, 5]  →  [5, 4, 3, 2, 1]
["a", "b", "c"]  →  ["c", "b", "a"]
[]                →  []
Tehnica cu doi pointeri: swap elementele de la index i cu cele de la size - 1 - i, mergând până la jumătate.
public static <T> void reverseList(List<T> list) {
    int n = list.size();
    for (int i = 0; i < n / 2; i++) {
        T temp = list.get(i);
        list.set(i, list.get(n - 1 - i));
        list.set(n - 1 - i, temp);
    }
}
// O(n) timp, O(1) spațiu
L-02 Eliminare duplicate păstrând ordinea Easy

Cerință

Dată o List<Integer>, returnează o nouă listă fără duplicate, păstrând prima apariție. Nu folosi Stream.distinct().

// Input:  [3, 1, 4, 1, 5, 9, 2, 6, 5, 3]
// Output: [3, 1, 4, 5, 9, 2, 6]
Iterează cu un HashSet<Integer> seen — adaugi în lista nouă doar dacă seen.add(elem) returnează true.
public static List<Integer> removeDuplicates(List<Integer> list) {
    Set<Integer> seen = new HashSet<>();
    List<Integer> result = new ArrayList<>();
    for (Integer elem : list) {
        if (seen.add(elem)) {
            result.add(elem);
        }
    }
    return result;
}
// One-liner: new ArrayList<>(new LinkedHashSet<>(list))
L-03 Partition List cu Predicate Medium

Cerință

Scrie o metodă generică ce primește o List<T> și un Predicate<T> și returnează două liste: [matching, nonMatching].

// Input:  [1, 2, 3, 4, 5, 6], predicate: n -> n % 2 == 0
// Output: [[2, 4, 6], [1, 3, 5]]
Creează două ArrayList-uri, iterează și adaugă fiecare element după predicate.test(item).
public static <T> List<List<T>> partition(
        List<T> list, Predicate<T> pred) {
    List<T> yes = new ArrayList<>();
    List<T> no  = new ArrayList<>();
    for (T item : list) {
        (pred.test(item) ? yes : no).add(item);
    }
    return List.of(yes, no);
}
L-04 Merge Sort pe LinkedList Hard

Cerință

Implementează Merge Sort pe o LinkedList<Integer> fără conversie la array. Nu folosi Collections.sort(). Complexitate O(n log n).

// Input:  [38, 27, 43, 3, 9, 82, 10]
// Output: [3, 9, 10, 27, 38, 43, 82]
Split cu subList(0, mid) și subList(mid, size), copiază în LinkedList-uri noi, aplică recursiv, merge cu doi iteratori.
public static LinkedList<Integer> mergeSort(LinkedList<Integer> list) {
    if (list.size() <= 1) return list;
    int mid = list.size() / 2;
    LinkedList<Integer> left  = new LinkedList<>(list.subList(0, mid));
    LinkedList<Integer> right = new LinkedList<>(list.subList(mid, list.size()));
    left = mergeSort(left);
    right = mergeSort(right);
    return merge(left, right);
}

private static LinkedList<Integer> merge(
        LinkedList<Integer> a, LinkedList<Integer> b) {
    LinkedList<Integer> res = new LinkedList<>();
    Iterator<Integer> itA = a.iterator(), itB = b.iterator();
    Integer vA = itA.hasNext() ? itA.next() : null;
    Integer vB = itB.hasNext() ? itB.next() : null;
    while (vA != null && vB != null) {
        if (vA <= vB) { res.add(vA); vA = itA.hasNext() ? itA.next() : null; }
        else          { res.add(vB); vB = itB.hasNext() ? itB.next() : null; }
    }
    if (vA != null) res.add(vA);
    if (vB != null) res.add(vB);
    itA.forEachRemaining(res::add);
    itB.forEachRemaining(res::add);
    return res;
}
S-01 Operații pe Mulțimi: Union, Intersection, Difference Easy

Cerință

Implementează 3 metode statice generice: union, intersection, difference.

A = {1, 2, 3, 4}    B = {3, 4, 5, 6}
union      → {1, 2, 3, 4, 5, 6}
intersect  → {3, 4}
difference → {1, 2}
Creează un nou HashSet din a, apoi: addAll(b) = union, retainAll(b) = intersect, removeAll(b) = difference.
public static <T> Set<T> union(Set<T> a, Set<T> b) {
    Set<T> r = new HashSet<>(a); r.addAll(b); return r;
}
public static <T> Set<T> intersection(Set<T> a, Set<T> b) {
    Set<T> r = new HashSet<>(a); r.retainAll(b); return r;
}
public static <T> Set<T> difference(Set<T> a, Set<T> b) {
    Set<T> r = new HashSet<>(a); r.removeAll(b); return r;
}
S-02 Custom Object în HashSet — equals / hashCode Medium

Cerință

Creează Student(int id, String name, double gpa). Egalitate pe id. Override equals() și hashCode(), demonstrează în HashSet.

Set<Student> s = new HashSet<>();
s.add(new Student(1, "Ana", 9.5));
s.add(new Student(1, "Ana Maria", 9.8)); // same id!
s.add(new Student(2, "Bogdan", 8.7));
// s.size() == 2
Contract: a.equals(b) → a.hashCode() == b.hashCode(). hashCode bazat pe id: return Integer.hashCode(id);
public class Student {
    private int id;
    private String name;
    private double gpa;
    // constructor, getters...

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        return this.id == ((Student) o).id;
    }

    @Override
    public int hashCode() {
        return Integer.hashCode(id);
    }
}
S-03 Primul caracter ne-repetat — LinkedHashMap Easy

Cerință

Dat un string, găsește primul caracter care apare o singură dată. Fără Streams.

"aabccbdefe" → 'd'
"aabbcc"     → null
LinkedHashMap<Character, Integer> pentru frecvențe (păstrează ordinea inserției), apoi iterează și returnează primul cu count == 1.
public static Character firstUnique(String s) {
    Map<Character, Integer> freq = new LinkedHashMap<>();
    for (char c : s.toCharArray())
        freq.merge(c, 1, Integer::sum);
    for (var e : freq.entrySet())
        if (e.getValue() == 1) return e.getKey();
    return null;
}
S-04 TreeSet — Interval Scheduling (Calendar) Hard

Cerință

Cu record Interval(int start, int end) și un TreeSet, implementează:

  • boolean book(Interval i) — adaugă dacă NU se suprapune
  • Set<Interval> getBookings() — toate, sortate
book([10,20]) → true
book([15,25]) → false  // overlap
book([20,30]) → true
book([5,10])  → true
Sortează după start. Folosește floor() și ceiling() din NavigableSet ca să verifici doar vecinii — O(log n).
record Interval(int start, int end) {}

public class Calendar {
    private TreeSet<Interval> bk =
        new TreeSet<>(Comparator.comparingInt(Interval::start));

    public boolean book(Interval i) {
        Interval lo = bk.floor(i);
        Interval hi = bk.ceiling(i);
        if (lo != null && lo.end() > i.start()) return false;
        if (hi != null && hi.start() < i.end()) return false;
        bk.add(i);
        return true;
    }
}
// O(log n) per book() datorită TreeSet
Q-01 Stack cu Deque — Verificare Paranteze Easy

Cerință

Verifică dacă un string are paranteze echilibrate: ()[]{}. Folosește ArrayDeque (nu Stack legacy).

"({[]})" → true     "([)]" → false
""       → true     "((("  → false
Push paranteza de închidere corespunzătoare. La closing bracket, verifică top == ea. La final, stack gol = valid.
public static boolean isBalanced(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '[') stack.push(']');
        else if (c == '{') stack.push('}');
        else if (c == ')' || c == ']' || c == '}') {
            if (stack.isEmpty() || stack.pop() != c) return false;
        }
    }
    return stack.isEmpty();
}
Q-02 PriorityQueue — Top K Elemente Medium

Cerință

Dat un array și k, returnează cele mai mari k elemente cu PriorityQueue (min-heap de size k). Complexitate O(n log k).

Input:  [3, 1, 5, 12, 2, 11, 7], k=3
Output: [7, 11, 12]
Min-heap de size k. Dacă size < k → add. Dacă elem > peek() → poll + add.
public static List<Integer> topK(int[] arr, int k) {
    PriorityQueue<Integer> heap = new PriorityQueue<>(k);
    for (int n : arr) {
        if (heap.size() < k) heap.offer(n);
        else if (n > heap.peek()) { heap.poll(); heap.offer(n); }
    }
    List<Integer> res = new ArrayList<>(heap);
    Collections.sort(res);
    return res;
}
// O(n log k) — mai bun decât O(n log n) sort complet
Q-03 Sliding Window Maximum — Deque O(n) Hard

Cerință

Dat un array și fereastră k, returnează maximul din fiecare fereastră. O(n).

Input:  [1, 3, -1, -3, 5, 3, 6, 7], k=3
Output: [3, 3, 5, 5, 6, 7]
// [1,3,-1] [3,-1,-3] [-1,-3,5] [-3,5,3] [5,3,6] [3,6,7]
ArrayDeque<Integer> cu indecși, descrescător (front = max). Elimină din spate valorile ≤ noul elem. Elimină din față indecșii ieșiți din fereastră.
public static int[] maxSlidingWindow(int[] nums, int k) {
    int[] res = new int[nums.length - k + 1];
    Deque<Integer> dq = new ArrayDeque<>();
    for (int i = 0; i < nums.length; i++) {
        while (!dq.isEmpty() && dq.peekFirst() < i - k + 1)
            dq.pollFirst();
        while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i])
            dq.pollLast();
        dq.offerLast(i);
        if (i >= k - 1)
            res[i - k + 1] = nums[dq.peekFirst()];
    }
    return res;
}
// Fiecare element intră/iese din deque maxim o dată → O(n)
Q-04 Task Scheduler — PriorityQueue + Cooldown Hard

Cerință

Task-uri (caractere) cu cooldown n între execuțiile aceluiași task. Returnează timpul minim total.

Input:  tasks=['A','A','A','B','B','B'], n=2
Output: 8
// A → B → idle → A → B → idle → A → B
Greedy: mereu execută task-ul cu cele mai multe apariții care nu e în cooldown. Max-heap pe frecvențe + Queue pentru cooldown tracking.
public static int leastInterval(char[] tasks, int n) {
    int[] freq = new int[26];
    for (char t : tasks) freq[t - 'A']++;

    PriorityQueue<Integer> maxH =
        new PriorityQueue<>(Collections.reverseOrder());
    for (int f : freq) if (f > 0) maxH.offer(f);

    Queue<int[]> cooldown = new LinkedList<>();
    int time = 0;

    while (!maxH.isEmpty() || !cooldown.isEmpty()) {
        time++;
        if (!maxH.isEmpty()) {
            int rem = maxH.poll() - 1;
            if (rem > 0) cooldown.offer(new int[]{rem, time + n});
        }
        if (!cooldown.isEmpty() && cooldown.peek()[1] == time)
            maxH.offer(cooldown.poll()[0]);
    }
    return time;
}
X-01 Two Sum — HashMap lookup O(n) Easy

Cerință

Dat un array și un target, returnează indicii a două elemente care însumează target-ul. O(n).

Input:  nums=[2, 7, 11, 15], target=9
Output: [0, 1]  // 2 + 7 = 9
Pentru fiecare num, caută target - num în map. Dacă nu există, pune map.put(num, index).
public static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int comp = target - nums[i];
        if (map.containsKey(comp))
            return new int[]{map.get(comp), i};
        map.put(nums[i], i);
    }
    throw new IllegalArgumentException("No solution");
}
X-02 Grupare Anagrame Medium

Cerință

Dată o listă de stringuri, grupează anagramele.

Input:  ["eat","tea","tan","ate","nat","bat"]
Output: [["eat","tea","ate"], ["tan","nat"], ["bat"]]
Sortează literele → "eat" → "aet". Asta devine cheia de grupare.
public static List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] ch = s.toCharArray();
        Arrays.sort(ch);
        map.computeIfAbsent(new String(ch), k -> new ArrayList<>())
           .add(s);
    }
    return new ArrayList<>(map.values());
}
X-03 FlattenIterator — Iterator custom lazy Hard

Cerință

Implementează Iterator<T> care primește List<List<T>> și iterează flat, lazy (fără pre-flatten).

List<List<Integer>> nested = List.of(
    List.of(1,2), List.of(), List.of(3), List.of(4,5,6));
// next() → 1, 2, 3, 4, 5, 6
Doi iteratori: outer (pe liste) și inner (pe elementele listei curente). hasNext() avansează la sublista non-vidă următoare.
public class FlattenIterator<T> implements Iterator<T> {
    private final Iterator<List<T>> outer;
    private Iterator<T> inner;

    public FlattenIterator(List<List<T>> lists) {
        outer = lists.iterator();
        inner = Collections.emptyIterator();
    }

    @Override
    public boolean hasNext() {
        while (!inner.hasNext() && outer.hasNext())
            inner = outer.next().iterator();
        return inner.hasNext();
    }

    @Override
    public T next() {
        if (!hasNext()) throw new NoSuchElementException();
        return inner.next();
    }
}
// Zero alocare extra — lazy evaluation!
X-04 Alege structura potrivită — Design Decisions Medium

Cerință

Pentru fiecare scenariu, alege implementarea optimă și justifică:

  • 1. Log events — append-only, citire secvențială
  • 2. Autocompletare — prefix search pe cuvinte
  • 3. Job scheduler — procesare pe prioritate
  • 4. Undo/Redo în editor de text
  • 5. IP-uri unice sortate pentru firewall

3. PriorityQueue<Job> + Comparator

4. Două ArrayDeque-uri (undoStack, redoStack)

   → O(1) next/prev cu ListIterator
   → O(1) insert/remove la poziția curentă
